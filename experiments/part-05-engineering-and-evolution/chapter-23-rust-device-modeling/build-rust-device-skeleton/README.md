# Build a Rust device skeleton

Status: runnable safe Rust register-state crate; QOM/SysBus integration remains
a separate tree-out exercise.

Target release: QEMU `v11.1.0`; source-review baseline `v11.1.0-rc0`; target machine
context RISC-V `riscv64`.

## Purpose

Design a minimal Rust sysbus device for the RISC-V `virt` machine with one MMIO
register, reset behavior, and a testable ownership model.

## Prerequisites

- Rust 1.83 or newer and Cargo.
- QEMU Rust build dependencies and a disposable worktree for the optional
  integration stage; familiarity with QOM, SysBus, and MemoryRegion.

## Files

- `README.md`: design and implementation procedure.
- `Cargo.toml`: dependency-free, non-published lab crate.
- `Cargo.lock`: pinned Cargo resolution generated by the tested toolchain.
- `src/lib.rs`: register, access validation, reset, and four unit tests.
- `qemu-integration.md`: explicit mapping to QEMU Rust APIs and lifecycle.

## Steps

1. Run `cargo fmt --check`, `cargo clippy --all-targets -- -D warnings`, and
   `cargo test` in this directory.
2. Read the register constants, validation function, state methods, and tests;
   explain which invariants Rust checks and which are protocol decisions.
3. Study QEMU's `rust/qom`, `rust/system`, `rust/migration`, and `rust/hw/core`
   APIs listed in `qemu-integration.md`.
4. In a disposable QEMU branch, add QOM/SysBus registration and one MMIO
   region, then wire it into a local RISC-V `virt` variant as separate patches.
5. Preserve the safe state tests, then add qtest reset and invalid-access tests
   before making any IRQ, DMA, or migration claim.

## Expected results

Four unit tests pass and Clippy accepts the safe crate. The optional QEMU stage
must expose the same deterministic register while keeping raw-pointer and QOM
lifetime assumptions in a small reviewed adapter.

## Cleanup

Run `cargo clean`, remove the exercise build, and discard the temporary QEMU
branch after preserving reviewed patches.

## Troubleshooting

- QEMU's Rust surface evolves rapidly; use only APIs present in the anchor.
- A successful Rust compile does not validate QOM lifetime assumptions.
- The committed crate deliberately does not claim IRQ, DMA, or migration
  support; those contracts need QEMU integration and system tests.
